
# HYBE Audit Red Flag & Interview Agent

## 1. 프로젝트 개요

본 프로젝트는 2023~2025년 HYBE 연결재무정보를 바탕으로
계정 간 관계와 추세를 분석하고, 추가 검토가 필요한 감사 위험 신호와
기업 담당자 인터뷰 질문을 제시하는 Python 기반 MVP입니다.

## 2. 해결하려는 문제

재무제표 검토 과정에서는 개별 숫자의 증감보다
매출과 수익성, 이익과 현금흐름, 수익성과 영업권,
매출과 운전자본 등 계정 간 관계가 더 중요한 단서를 제공할 수 있습니다.

본 MVP는 이러한 관계를 사전에 정의한 규칙으로 계산하고,
다음 정보를 하나의 Trusted Output으로 제공합니다.

- 확인된 재무 사실
- 적용된 관계형 규칙
- 규칙 충족 여부
- 가능한 정상적 설명
- 가능한 위험 해석
- 경영진 인터뷰 질문
- 요청할 증빙
- 후속 검토 행동
- 근거 강도와 데이터 완전성

## 3. 분석 범위

- 대상 기업: HYBE
- 재무제표: 연결재무제표
- 분석기간: 2023~2025년
- 단위: 백만원
- 전체 분석 규칙: 10개
- 규칙 충족 Red Flag: 8개
- Monitoring Signal: 2개

## 4. 주요 기능

1. pandas 기반 재무데이터 입력 및 검증
2. 계정 간 증감률과 비율 자동 계산
3. 결정론적 Red Flag 규칙 판정
4. Red Flag와 Monitoring Signal 분리
5. 경영 관점과 감사 관점 분리
6. 관련 계정과 재무제표 주장 매핑
7. 인터뷰 질문과 요청 증빙 생성
8. 중복 위험군 제거 후 Top 3 선정
9. 외부 공식 사례를 활용한 블라인드 구조 검증
10. JSON, CSV, Markdown 결과물 저장

## 5. 시스템 구조

공식 재무정보
→ 입력값 검증
→ Python 관계 계산
→ 사전 정의 규칙 판정
→ Red Flag / Monitoring Signal 분류
→ 위험군별 중복 제거
→ 인터뷰 질문, 증빙, 후속 행동
→ Trusted Output Report

## 6. 외부 검증

기존 10개 규칙을 먼저 동결한 뒤,
공식 공시에 기재된 건설계약 관련 회계오류 사례 1건을
익명 블라인드 사례로 사용했습니다.

- 검증 요소 포착: 7/8
- 요소 포착률: 87.5%
- 건설계약 특화 규칙 포착: False

본 수치는 정확도나 오류 탐지 확률이 아닙니다.
사전에 정의한 검증 요소 중 기존 규칙, 질문 및 증빙 구조가
포함한 요소의 비율입니다.

일반적인 매출채권, 재고자산 위험은 포착했지만,
건설계약 진행률 및 미청구공사 등 업종 특화 위험은
포착하지 못했습니다.

## 8. 현재 한계

- 일부 재무수치는 공식 공시 원문과 최종 재대조가 필요합니다.
- 사업부문별 손익과 원장 수준 데이터는 포함하지 않았습니다.
- 현금창출단위별 손상평가 자료와 주요 계약서는 확인하지 않았습니다.
- 외부 검증 사례가 1건이므로 일반화 성능을 입증하지 못합니다.
- 현재 가설과 질문은 생성형 AI API가 아닌 설계자가 작성한 MVP입니다.
- 우선순위 점수는 회계오류나 부정의 발생 확률이 아닙니다.

## 9. 향후 개선 방향

- DART 공시 데이터 자동 수집
- 업종별 위험 규칙 모듈
- 생성형 AI API 기반 가설 및 질문 생성
- 근거 문서 검색과 인용 자동화
- 사용자 피드백 기반 질문 개선
- 외부 사례 확대 및 오탐 분석
- Streamlit 또는 웹 기반 인터페이스

## 10. 주의사항

식별된 Red Flag는 오류나 부정의 존재를 의미하는 것이 아니라,
기업 및 환경에 대한 이해와 중요왜곡표시위험 평가를 위해
추가적인 질문과 검토가 필요한 영역을 의미합니다.


## 범용 재무제표 업로드 MVP

Colab Notebook에서 새로운 CSV 파일을 업로드하면 다음 절차를 자동 실행합니다.

1. 계정명 열과 연도 열 자동 인식
2. 한국어 및 영문 계정명 표준화
3. 필수 계정, 중복 계정, 숫자 오류 검증
4. 10개 결정론적 감사 위험 규칙 실행
5. Red Flag와 Monitoring Signal 분류
6. 규칙별 인터뷰 질문 및 요청 증빙 연결
7. Top 3 우선 검토 영역 선정
8. JSON 및 Markdown 보고서 저장

현재 질문과 위험 가설은 규칙별 사전 정의 템플릿입니다.
생성형 AI API는 아직 연결하지 않았습니다.

식별된 Red Flag는 오류나 부정의 존재를 의미하는 것이 아니라, 기업 및 환경에 대한 이해와 중요왜곡표시위험 평가를 위해 추가적인 질문과 검토가 필요한 영역을 의미한다.
